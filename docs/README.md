# Documentation

Reference materials, API specifications, and architectural documentation.

## Files

- **schema.md** — Database and Google Sheet schema definitions
- **api-reference.md** — External API documentation (OSRM, Gemini, Sheets, etc.)
- **architecture.md** — System architecture and data flow diagrams
- **ARCHITECTURE.md** — Detailed architecture decisions and patterns

## When to Use

Read these files when:
- Building new features that interact with external APIs
- Understanding data structure and field meanings
- Reviewing architectural decisions and their rationale
- Troubleshooting integration issues

## How to Contribute

If you discover new APIs, schema changes, or architectural patterns:
1. Update the relevant file here
2. Note the date of change
3. Cross-reference in `skills.md` if it's a reusable pattern
